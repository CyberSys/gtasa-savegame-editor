Vehicle

0123 4567 89AB CDEF 0123 4567 89AB CDEF
0000 ID Type C1 C2 X Y Z H Alm Lck FL 0000 Health 0000 Timer FFFF Avail 00 N
0020 0000
  • The ID of a vehicle should be one more than the ID of the previous vehicle. The first vehicle has ID 0x20.
  • Type indicates the vehicle type ID.
  • C1 is the color of the vehicle, use -1 for random.
  • C2 is the stripe color, use -1 for random.
  • X, Y and Z are the coordinates of the vehicle multiplied by 8.
  • H is the heading: 0 = North, 64 = West, 128 = South, 192 = East
  • Alm is the chance of an alarm. (%)
  • Lck is the chance of being locked. (%)
  • FL is the flags.
    Use 0x10 as default flag. (only bit 4 enabled)
    • Bit 0 means the car has moved and is waiting to respawn.
    • Bit 1 means the vehicle belongs to the player.
    • Bit 2 set to 1 crashed the game.
    • Bit 3 mean probably it's a special vehicle. (the two cars & the BMX in Grove St are flag 24)
    • Bit 4 seems to always be 1.
    • Bit 5, 6 and 7 had no effect.
  • Health is the health of the car when it spawns (is always 10,000)
    Changing this value has no effect.
  • If Timer is less than the global timer, it is the time when the vehicle last appeared.
    If greater than the global timer (bit 0 of the flags must be set to 1), it is the time when it will re-spawn.
  • Avail should be FF FF when available and 00 00 when not available.
  • N is the number of cars to generate, usually always 1.

  • Number Plate is an ASCII string. There shouldn't be a 00 byte in it. Spaces are used for padding.
